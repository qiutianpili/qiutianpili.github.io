Vim 编辑器核心命令指南

高效开发、极速运维与 Linux 终端文本处理的必备手册

在日常管理 Linux（如 Debian）服务器、编写自动化脚本或是处理复杂配置文件时，Vim 凭借其高度的灵活性和纯键盘操作的特性，成为了不可或缺的神器。本指南基于经典的 Vim 学习路径，为您总结了最常用的核心命令。

一、 Vim 的三种基本模式

理解 Vim 的关键在于掌握它的三种核心模式，在终端中启动 Vim 后（如输入 vim filename），默认进入的是一般命令模式。

🛡️ 一般命令模式 (Normal Mode)

启动时的默认模式。用于移动光标、复制、粘贴、删除文本。在这个模式下，敲击的任何按键都会被视作指令。

✍️ 编辑模式 (Insert Mode)

按下 i, I, o, O, a, A 等按键进入。在此模式下才可以输入文字。按 Esc 键退出回到一般模式。

⚙️ 命令行模式 (Command-line Mode)

在一般模式下按下 :, /, ? 进入。用于保存文件、退出 Vim、显示行号或进行全局搜索替换。

二、 核心命令速查表

1. 光标移动 (一般模式)

命令功能说明
h / j / k / l左 / 下 / 上 / 右 移动光标（推荐替代方向键）
Ctrl + f / Ctrl + b向下翻一页 (Forward) / 向上翻一页 (Backward)
0 或 Home移动到当前行的最前面
$ 或 End移动到当前行的最后面
G移动到这个文件的最后一行
gg移动到这个文件的第一行
nG移动到这个文件的第 n 行（如 20G 移动到第 20 行）

2. 文本插入 (进入编辑模式)

命令功能说明
i / Ii 在当前光标前插入；I 在当前行的非空格符处插入
a / Aa 在当前光标后插入；A 在当前行最后插入
o / Oo 在当前行下方插入新行；O 在当前行上方插入新行

3. 复制、粘贴与删除 (一般模式)

命令功能说明
x / Xx 向后删除一个字符；X 向前删除一个字符
dd删除（剪切）游标所在的那一整行
ndd删除（剪切）向下 n 行（例如 10dd）
yy复制游标所在的那一行
nyy复制向下 n 行（例如 20yy）
p / Pp 在光标下一行粘贴；P 在光标上一行粘贴
u复原前一个动作 (Undo)
Ctrl + r重做上一个动作 (Redo)

4. 搜索与替换 (命令行模式)

命令功能说明
/word向光标之下寻找名为 word 的字符串。按 n 继续寻找下一个。
?word向光标之上寻找名为 word 的字符串。
:n1,n2s/word1/word2/g在第 n1 到 n2 行之间寻找 word1，并替换为 word2
:1,$s/word1/word2/g从第一行到最后一行寻找 word1，并替换为 word2 (全局替换)

5. 文件保存与退出 (命令行模式)

命令功能说明
:w将编辑的数据写入硬盘档案中
:w!若文件属性为只读，强制写入该文件
:q退出 Vim (未修改文件时)
:q!强制退出 Vim，放弃所有修改
:wq 或 :x保存并退出
:set nu / :set nonu显示行号 / 取消显示行号
💡 效率提示： 在处理繁重的开发项目或者排查长篇日志时，熟练运用 / 搜索以及 gg 和 G 可以极大地提升文件导航效率。遇到意外修改，不要慌张，多按几次 Esc 确保回到一般模式，然后按下 u 即可撤销。
