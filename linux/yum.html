Linux yum 命令核心指南

CentOS/RHEL 等红帽系发行版的高效包管理工具

yum (Yellow dog Updater, Modified) 是在 Fedora、RedHat 以及 CentOS 中的 Shell 前端软件包管理器。它基于 RPM 包管理，能够从指定的服务器自动下载 RPM 包并且安装。其最强大的特性在于能够自动处理依赖性关系，一次安装所有依赖的软件包，无须繁琐地一次次下载和安装。

一、 yum 的核心优势

🔗 自动解决依赖

在安装软件时，自动计算并下载该软件运行所需的其他依赖包，完美解决传统 RPM 安装时的 "依赖地狱" 问题。

📦 软件源 (Repo) 管理

支持配置多个资源库 (Repository)，通过修改配置文件可以轻松切换国内镜像源（如阿里云、清华源）以提升下载速度。

🛠️ 简洁的命令设计

语法直观，提供搜索、安装、更新、卸载一站式服务，极大地简化了 Linux 系统管理员的日常运维工作。

二、 核心命令速查表

1. 软件安装与更新

命令功能说明
yum install <package>安装指定的软件包（追加 -y 参数自动回答 yes）
yum update更新系统中所有已安装的软件包
yum update <package>仅更新指定的软件包
yum check-update列出所有可更新的软件清单
yum upgrade大规模的版本升级（连同废弃的包一起升级）

2. 查找与显示信息

命令功能说明
yum list列出所有可安裝的软件清单
yum list installed仅列出系统中已经安装的软件包
yum search <keyword>在软件源中搜索包含指定关键字的软件包
yum info <package>显示指定软件包的详细信息（版本、大小、描述等）
yum provides <command>查找提供指定命令（或文件）的软件包（例如：yum provides ifconfig）

3. 卸载与缓存清理

命令功能说明
yum remove <package>卸载指定的软件包（也会同时卸载依赖该包的其他软件，需谨慎）
yum clean all清除所有的缓存目录下的软件包及旧的 headers
yum makecache将服务器上的软件包信息缓存到本地，以提高搜索和安装速度

4. 软件组管理 (Group)

命令功能说明
yum grouplist列出所有可用的软件组（如 "Development Tools"）
yum groupinstall "<group>"安装指定的软件组（包含一组相关的软件，适合一键配置环境）
yum groupremove "<group>"卸载指定的软件组
💡 运维提示： 在配置新的 yum 软件源（修改 /etc/yum.repos.d/ 目录下的 .repo 文件）之后，务必依次执行 yum clean all 和 yum makecache 命令，以确保包管理器使用的是最新的元数据。另外，在 CentOS 8 及更新版本中，虽然底层已替换为更快的 dnf，但 yum 命令仍作为别名被保留并完全兼容。
